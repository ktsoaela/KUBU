Welcome to Kubu

Kubu is a CSS-focused frontend framework that uses pure CSS for styling and animations, with no reliance on JavaScript. It adopts BEM (Block Element Modifier) and Atomic Design principles to create modular, reusable, and scalable components. For example, the accordion component, built entirely with CSS, allows you to toggle the visibility of content sections in a collapsible format. By leveraging Atomic Design, Kubu structures UI components as atoms, molecules, and organisms, ensuring clear, manageable, and highly reusable code. BEM further enhances this by providing a consistent naming convention for components, their elements, and their variations.

How BEM and Atomic Design Can Work Together

  1. Atomic Design as the Component Hierarchy:

    Atomic Design structures UI into a hierarchy of components: atoms (smallest elements), molecules (combinations of atoms), organisms (complex UI structures), templates (overall layouts), and pages (final user views). This structure allows you to categorize components based on their reusability and function within the design system.

  2. BEM for Naming and Structure:

    BEM's naming convention ensures clear relationships between components and their variations, aiding readability and reusability. Each component in Atomic Design can be named according to BEMâ€™s structure:

    • Block: represents a standalone component (e.g., card, button).
    • Element: parts within a component (e.g., card__image, button__icon).
    • Modifier: represents variations of a component or element (e.g., button--primary, card--compact).
  3. Example Integration:

    For an Accordion component (a molecule), the naming could look like this:

    
.accordion {
  /* Base styles for the accordion molecule */
}

.accordion__header {
  /* Header element inside the accordion */
}

.accordion__header--expanded {
  /* Modified state for an expanded accordion header */
}

.accordion__content {
  /* Content element inside the accordion */
}

    By organizing CSS like this, each componentâ€™s elements and modifiers remain distinct, clear, and manageable. This approach works well in Atomic Design since each level of component hierarchy (atoms, molecules, etc.) can have its own BEM naming structure.

  4. Advantages:
    • Scalability: Both BEM and Atomic Design promote a scalable system where changes at any level wonâ€™t disrupt the entire project.
    • Clarity and Consistency: Each part of the UI has a clear naming pattern and position in the hierarchy.
    • Reusability: Components and styles are highly reusable, with variations handled cleanly through BEM modifiers.

Applying this Combination

Using BEM and Atomic Design together in a CSS framework like Kubu would mean each component is organized by functionality and styled with clear, consistent class names. For instance, in a project where button styles are reused across multiple contexts, creating button atoms with BEM names (button--primary, button--secondary) ensures that they can be easily customized without duplicating code.
